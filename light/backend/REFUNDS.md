# Refunds — Light Module

This document describes the minimal refund functionality implemented in the Light module (backend + frontend).

## Overview
- Endpoint: `POST /api/pos/transactions/{transaction_id}/refund`
- Purpose: perform a minimal refund for a POS transaction, optionally restock items, and persist an audit record.
- Authorization: requires a Manager account (route depends on `get_current_manager`).

## Request (JSON)
- `amount` (number, required) — refund amount (must be > 0 and <= original transaction total)
- `reason` (string, optional) — textual reason for the refund
- `restock` (boolean, optional, default: false) — if true, increases inventory quantities for the refunded items
- `refund_payment_method` (string, optional) — payment method for the refund (e.g., `cash`, `card`)

Example request body:

```json
{
  "amount": 10.00,
  "reason": "Customer returned item",
  "restock": true
}
```

## Response (JSON)
- `transaction_id` — ID of the original POS transaction
- `refunded_amount` — amount refunded
- `refund_date` — timestamp of refund (ledger record timestamp)
- `new_balance` — ledger balance after applying the refund (returns the ledger record balance)
- `restocked` — whether items were restocked
- `notes` — optional notes from the ledger entry

Example response:

```json
{
  "transaction_id": 123,
  "refunded_amount": 10.0,
  "refund_date": "2026-03-27T12:34:56.789Z",
  "new_balance": 990.0,
  "restocked": true,
  "notes": "Customer returned 1 item"
}
```

## Behavior
- Validates the refund amount against the original transaction total.
- Prevents duplicate refunds: the same transaction can only be refunded once (second attempt returns `409 Conflict`).
- If `restock` is true, increments inventory quantities for items belonging to the transaction.
- Persists a `LedgerRecord` with `transaction_type = RETURN` and adjusts the running balance.
- Persists a `Refund` database row in the `refunds` table (model defined in `models.py`).
- The database tables are created automatically during application startup via `init_db()` (calls `Base.metadata.create_all`).

## DB Model
- The `Refund` model lives in `light/backend/models.py` and contains: `transaction_id`, `amount`, `refund_date`, `reason`, `restocked`, `processed_by`, `ledger_id`, timestamps.
